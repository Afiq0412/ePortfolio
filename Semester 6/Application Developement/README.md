# Application Development

## Course Overview

This course helped me understand how an application grows from an idea into a usable system. The work in this folder includes proposal and module documents that show planning, requirements, and development thinking.

## What I Gained

I learned to think more carefully about the user, the system flow, and the purpose behind each feature. Instead of only focusing on code, I practiced connecting requirements, interface behavior, and project documentation into one clearer development process.

## Skills Practiced

| Area | Growth |
| --- | --- |
| Requirement planning | Translating a problem into application features and modules |
| Documentation | Preparing proposals, details, and supporting reports |
| User-focused thinking | Considering how the application should be useful and understandable |
| Team/project work | Organizing development tasks around a shared goal |

## Reflection

This course made me more aware that good applications are not built by coding immediately. They need planning, clear scope, and decisions that match real user needs. I gained confidence in explaining an application idea and preparing the materials needed before implementation.
